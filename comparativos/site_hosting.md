# Hospedagem de Sites Simples

Comparativo rápido da dferença entre os três "modelos" de criação/hospedagem de sites simples.

*Abra este documento em um visualização Markdown melhor que o do Github se quiser ter uma experiência mais "colorida".*

| Critério                      | Editor online (tipo Wix etc)                       | WordPress (ou similar) em servidor próprio ou locado    | Site manual (HTML/CSS/JS/framework) em servidor próprio ou locado |
| ----------------------------- | -------------------------------------------------- | ------------------------------------------------------- | ----------------------------------------------------------------- |
| Velocidade para colocar no ar | <span style="font-weight:bold;color:green;">Muito alta</span> (horas, poucos dias)                | <span style="font-weight:bold;color:yellow;">Média</span> (dias, às vezes semanas)                      | <span style="font-weight:bold;color:orange;">Baixa a Média</span> (dias a semanas, depende do escopo)             |
| Facilidade para leigos        | <span style="font-weight:bold;color:green;">Muito alta</span>                                     | <span style="font-weight:bold;color:yellow;">Média</span> (exige noção de CMS e plugins)                | <span style="font-weight:bold;color:red;">Baixa</span> (exige dev)                                             |
| Custo inicial                 | <span style="font-weight:bold;color:green;">Baixo a médio</span>                                      | <span style="font-weight:bold;color:yellow;">Médio</span>                                                   | <span style="font-weight:bold;color:orange;">Médio a alto</span>                                                      |
| Custo mensal                  | <span style="font-weight:bold;color:orange;">Assinatura da plataforma</span> (pode subir com recursos) | <span style="font-weight:bold;color:yellow;">Hospedagem + manutenção</span> (variável)                      | <span style="font-weight:bold;color:yellow;">Hospedagem + manutenção</span> (variável)                                |
| Liberdade de design           | <span style="font-weight:bold;color:yellow;">Média</span> (limitado ao editor/tema)                    | <span style="font-weight:bold;color:green;">Alta</span> (temas, page builders, custom)                     | <span style="font-weight:bold;color:green;">Muito alta</span> (você faz tudo)                                    |
| Flexibilidade técnica         | <span style="font-weight:bold;color:yellow;">Média</span>                                              | <span style="font-weight:bold;color:green;">Alta</span> (plugins, código, APIs)                            | <span style="font-weight:bold;color:green;">Muito alta</span> (controle total)                                   |
| Controle e propriedade        | <span style="font-weight:bold;color:red;">Menor</span> (plataforma dita regras e limites)       | <span style="font-weight:bold;color:green;">Alto</span>                                                    | <span style="font-weight:bold;color:green;">Máximo</span>                                                        |
| Risco de lock-in              | <span style="font-weight:bold;color:red;">Alto</span> (migração pode ser chata)                 | <span style="font-weight:bold;color:yellow;">Médio</span> (mais portável)                                   | <span style="font-weight:bold;color:green;">Baixo</span> (se bem estruturado)                                        |
| Desempenho (potencial)        | <span style="font-weight:bold;color:yellow;">Médio</span> (depende da plataforma)                      | <span style="font-weight:bold;color:orange;">Médio a alto</span> (depende de tema, cache, plugins)          | <span style="font-weight:bold;color:green;">Alto</span> (otimização sob medida)                                  |
| SEO (potencial)               | <span style="font-weight:bold;color:yellow;">Médio a bom</span> (recursos prontos, mas com limites)    | <span style="font-weight:bold;color:green;">Bom a muito bom</span> (bem configurado)                   | <span style="font-weight:bold;color:green;">Muito bom</span> (se feito com boas práticas)                        |
| Segurança                     | <span style="font-weight:bold;color:green;">Plataforma cuida da infra</span>, você cuida de acessos   | <span style="font-weight:bold;color:orange;">Você cuida</span> de atualizações, backups, hardening      | <span style="font-weight:bold;color:red;">Você cuida de tudo</span> (mais responsabilidade)                    |
| Manutenção                    | <span style="font-weight:bold;color:green;">Baixa</span> (a plataforma resolve quase tudo)            | <span style="font-weight:bold;color:orange;">Média a alta</span> (atualizações, conflitos de plugins)       | <span style="font-weight:bold;color:yellow;">Média</span> (depende do stack e disciplina de deploy)                   |
| Escalabilidade                | <span style="font-weight:bold;color:yellow;">Média</span> (limites e planos)                           | <span style="font-weight:bold;color:green;">Alta</span> (melhora com infra)                                | <span style="font-weight:bold;color:green;">Muito alta</span> (arquitetura sob medida)                           |
| Integrações e automações      | <span style="font-weight:bold;color:yellow;">Boas</span>, mas dentro do "ecossistema"                  | <span style="font-weight:bold;color:green;">Muito boas</span> (plugins + integrações)                      | <span style="font-weight:bold;color:green;">Máximas</span> (qualquer integração via código)                      |
| Melhor para                   | Landing pages, sites simples, rapidez              | Sites institucionais robustos, blog, marketing contínuo | Produto digital, site altamente custom, performance e controle    |
